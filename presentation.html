System Architecture
How the code is organized
Clean separation between frontend, API, database, and AI services.
┌─────────────────┐ HTTP / REST ┌──────────────────────────────┐ │ React SPA │ ◄──────────────────► │ Node.js + Express API │ │ (Vite + Tailwind)│ (JWT + Cookie) │ • Auth / Patients / Consults│ └─────────────────┘ │ • Appointments / Reports / AI│ │ • Scheduler / Mailer │ └────────────┬─────────────────┘ │ ┌────────────▼─────────────────┐ │ MySQL Database │ │ patients, users, consults │ │ appointments, clinic_reports│ └──────────────────────────────┘
Key decisions: JWT in httpOnly cookies (secure, no JS access), pluggable AI engine with local fallback
Production: Single port serves API + built React app. SPA fallback for all client-side routes.
Database Design
How data is stored
Five core tables with indexes for fast search and reporting.
TablePurpose
patientsDemographics, DOB, sex, address, emergency contact
usersStaff accounts (admin/doctor/nurse/staff) + patient portal accounts with lockout
consultationsVisit records with raw_notes, structured_notes, diagnosis, prescription
appointmentsScheduled visits with status: scheduled / completed / cancelled
clinic_reportsAI-generated daily/weekly reports + JSON metrics for analytics
Indexed columns: patients.last_name, consultations.visit_date, appointments.appointment_date, clinic_reports.report_date
Authentication & Security
How the code protects the system
Security is not an afterthought — it is built into every layer.
FeatureHow it works
JWT AuthenticationToken from httpOnly cookie or Authorization: Bearer header
Role-Based AccessRoles: admin, doctor, nurse, staff, patient — each route is gated
Rate LimitingStrict limits on login/register to prevent brute-force
Account LockoutFailed attempts increment a counter; account locks after threshold
Helmet + CSPProduction security headers + Content Security Policy
HSTSForces HTTPS for a full year in production
Graceful ShutdownSIGTERM drains requests, closes DB pool, force-exits after 10s timeout

AI Feature 1
AI Note Summarizer
How raw doctor notes become structured medical records automatically.
📝
Doctor's Workflow
One click to structure notes
Step 1
Doctor types raw notes in plain language during or after the consultation.
Doctor types
"35f fever 3d, cough, throat inflamed, amoxicillin 500 3x7d paracetamol prn"
Step 2
Clicks "Summarize with AI" — backend sends note to Google Gemini with a medical prompt template.
Step 3
AI returns structured JSON which is saved into the consultation record.
🛡️
Resilient Fallback Chain
Works even without internet
The AI engine is designed to never fail:
Primary
Google Gemini — cloud AI with highest quality
Fallback 1
Free AI (Groq) — fast, low-cost cloud alternative
Fallback 2
Local AI Parser — regex-based medical parser that works 100% offline
Config
USE_LOCAL_AI=true skips cloud entirely for offline clinics
AI Feature 2
Auto-Generated Clinic Reports
Daily and weekly summaries generated automatically.
📊
Automatic Daily Flow
Runs at 6 PM every day
Step 1 — Scheduler
Cron scheduler triggers generateDailyReport(date).
Step 2 — Analytics
buildAnalytics() computes totals, complaints, diagnoses, peak hour, trends, follow-ups.
Step 3 — AI Generation
Metrics sent to Gemini. Plain-language report saved to clinic_reports.
Step 4 — Delivery (optional)
If SMTP configured, report is emailed automatically to the clinic owner.
📝
Example AI Output
Plain language, actionable insights
Daily Report Example
"Today, June 14: 24 patients served. Peak hours were 9–11 AM. Most common complaints: fever (8), cough (5), hypertension follow-up (4)."
Weekly Report Example
"Week of June 8–14: 156 consultations. Monday busiest. New patients: 23. Top diagnosis: URTI (34)."
Value to owner
No spreadsheets, no manual counting, no Excel needed.
API & Backend Logic
How requests flow through the server
server.js mounts all routes under /api.
EndpointPurpose
POST /api/auth/loginJWT login (rate-limited, account lockout)
GET /api/patientsList / search patients with pagination
POST /api/consultations/:id/summarizeTrigger AI note summarization
GET /api/reports/dailyGenerate or retrieve daily report
GET /api/reports/weeklyGenerate or retrieve weekly report
GET /api/ai/...Direct AI endpoints for testing
GET /healthLiveness probe for Docker / orchestrators
GET /health/readyReadiness probe — checks DB connectivity
Production mode: Same port serves API + built React app from frontend/dist
SPA fallback: Unknown non-API routes redirect to index.html for React Router
